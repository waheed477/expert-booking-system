## BACKEND REQUIREMENTS (Node.js + Express + MongoDB + Socket.io):

### Database Models:

1. Expert Model:
   - name (String, required)
   - category (String, required: ['Astrology', 'Vastu', 'Numerology', 'Tarot'])
   - experience (Number, required)
   - rating (Number, default: 0, min:0, max:5)
   - totalReviews (Number, default: 0)
   - bio (String, required)
   - profileImage (String)
   - hourlyRate (Number, required)
   - languages (Array)
   - availableSlots: [{
       date: Date,
       slots: [{
         time: String,
         isBooked: Boolean,
         bookedBy: String
       }]
     }]

2. Booking Model:
   - bookingId (String, unique)
   - expertId (ObjectId, ref: 'Expert')
   - expertName (String)
   - userName (String, required)
   - userEmail (String, required)
   - userPhone (String, required)
   - bookingDate (Date, required)
   - timeSlot (String, required)
   - notes (String)
   - status: ['pending', 'confirmed', 'completed', 'cancelled']
   - amount (Number)
   - createdAt: Date

### API ENDPOINTS:

1. GET /api/experts?page=1&limit=6&category=Astrology&search=raj
2. GET /api/experts/:id
3. POST /api/bookings
4. PATCH /api/bookings/:id/status
5. GET /api/bookings?email=user@example.com

### CRITICAL BACKEND FEATURES:

1. DOUBLE BOOKING PREVENTION:
   - Use MongoDB unique compound index on {expertId, bookingDate, timeSlot}
   - Implement database transactions for atomic operations
   - Add race condition handling

2. REAL-TIME UPDATES:
   - Socket.io setup with rooms per expert
   - Emit 'slotBooked' event to all connected clients
   - Handle disconnection and reconnection

3. ERROR HANDLING:
   - Global error handler middleware
   - Input validation using Joi/express-validator
   - Meaningful error messages

   - Reconnect on disconnect

2. Events:
   - Listen for 'slot:booked'
   - Listen for 'booking:confirmed'
   - Emit 'slot:select' for temporary lock

3. UI Updates:
   - Immediately disable booked slots
   - Show toast notification
   - Update booking counts
